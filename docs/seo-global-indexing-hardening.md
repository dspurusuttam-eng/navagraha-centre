## 2) Locale + Route Patterns

- Live locales: `en`, `as`, `hi`.
- Default locale: `en`.
- Locale-prefixed URLs supported through proxy rewrite (`/en/*`, `/as/*`, `/hi/*`).
- Root URLs continue to work (`/` and existing non-prefixed marketing paths).

Primary SEO route groups:
- Core tools: `/kundli`, `/compatibility`, `/rashifal`, `/panchang`, `/numerology`, `/ai`, `/navagraha-ai`, `/reports`, `/consultation`, `/shop`.
- Blog authority desk: `/from-the-desk` and `/from-the-desk/[slug]`.
- Utility discovery: `/tools`, `/calculators`, `/muhurta`.

## 3) Canonical + Hreflang Strategy

- Canonical URLs are generated by metadata helpers based on locale + explicit locale-prefix context.
- Alternates are generated for all live locales (`en`, `as`, `hi`) plus `x-default`.
- Root remains indexable for default English access.
- Locale routes are also indexable for multilingual discovery.

## 4) Sitemap Strategy

- Includes root, localized core routes, tool pages, blog desk, and dynamic content.
- Includes:
  - static multilingual routes with alternates
  - daily Rashifal sign pages
  - shop product pages
  - content entries grouped by translation group
- Priorities:
  - home: `1.0`
  - core tools: `0.9`
  - desk landing: `0.8`
  - articles and product pages: `~0.7`
  - utility/support pages: `0.4 - 0.7`

## 5) Schema Strategy

Global:
- Organization
- WebSite
- LocalBusiness/ProfessionalService

Page-level (where relevant):
- WebPage
- Service
- BreadcrumbList
- Person (J P Sarmah authority surface)
- BlogPosting/Article for content pages
- FAQPage for FAQ-backed pages
- Product schema for shop pages when product data is available

Important rule:
- No fake ratings/reviews/availability claims.

## 6) Crawl and Index Control

robots rules allow public pages and disallow:
- `/api/`
- `/admin/`
- `/dashboard/`
- `/private/`
- preview/draft/internal/test/dev paths
- account/auth paths and cart/system surfaces

## 7) Validation Utilities

- i18n parity checker: `npm run i18n:check`
- SEO checker: `npm run seo:check`
- Standard checks: `npm run lint`, `npm run typecheck`, `npm run build`
